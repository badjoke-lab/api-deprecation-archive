# Report Corrections

API Deprecation Archive welcomes corrections and missing-record reports based on public evidence.

## Report types

Use the reporting path for:

- missing API deprecation
- wrong deadline
- wrong removal status
- wrong still-usable status
- new migration guide
- new replacement URL
- broken evidence link
- record correction
- provider correction

## Useful report details

A useful report should include:

- provider name
- API / SDK / CLI / feature name
- what changed
- official source URL
- deadline or removal date, if known
- replacement URL, if known
- whether it is already removed, if known
- optional reporter note

## Evidence requirements

Official sources are preferred. Archived official pages are acceptable when current official pages moved or disappeared.

Secondary sources may help provide context, but initial records should not rely only on unclear or unsourced claims.

## Safety limits

Do not submit exploit instructions, API key bypass methods, authentication workaround instructions, or private credentials.

## Review process

Reports are reviewed before data changes are made. The archive may keep uncertainty notes when provider documentation is ambiguous.
